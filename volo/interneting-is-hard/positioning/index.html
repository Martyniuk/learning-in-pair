Conclusion

  • Static - normal flow positioning
      Next properties have no effect:
    • top
    • right
    • bottom
    • left
    • z-index
  • Relative - normal flow positioning. (The containing block is formed by the edge of the content box of the nearest ancestor element)
    Can apply top, right, bottom, left props
    Creates new stacking context when z-index in not auto
  • Absolute - removed from the normal document flow (The containing block is formed by the edge of the padding box of the nearest ancestor element that has a position value other than static(fixed, absolute, relative, sticky))
    Can apply top, right, bottom, left props
    Creates new stacking context when z-index in not auto
  • Fixed - removed from the normal document flow. Positioned relative to the initial containing block (or element that has non default values for next properties - filter, perspective / transform, will-change, contain: paint)
    Can apply top, right, bottom, left props
    Above rules apply for Absolute positioning too.
  • Sticky - normal flow positioning. "Sticks" to its nearest ancestor that has a "scrolling mechanism"
    Can apply top, right, bottom, left props
    Always creates new stacking context!
